WordList

Praat 中的 对象类型 之一。 WordList 类的对象以系统无关的格式包含一个排序的字符串列表。 WordList 对象在转换为 SpellingChecker（拼写检查器） 对象后，可用于拼写检查。

1. 如何创建 WordList 对象

您通常会通过读取二进制 WordList file 来创建 WordList 对象。 您将使用 Open 菜单中的通用 从文件读取... 命令。

如何创建此类文件，请参阅下面的第 3 点。

2. 您可以使用 WordList 对象做什么

WordList 的主要功能是能够告诉您它是否包含某个特定的字符串。 如果您选中一个 WordList，您可以使用 Has word 命令查询是否存在特定单词。您输入单词并按下“确定”。如果 WordList 确实包含该单词， 数值 "1" 将被写入信息窗口；否则，将写入数值 "0"。

3. 如何创建二进制 WordList 文件

您可以从包含长单词列表的简单文本文件创建二进制 WordList 文件。 此类文本文件可能由您国家的辞书机构提供； 由于版权问题，此类单词列表无法随 Praat 程序一起分发。 要将简单文本文件转换为压缩的 WordList 文件，您基本上需要执行以下步骤：

Read Strings from raw text file: "lexicon.txt"
 Sort
 To WordList
 Save as binary file: "lexicon.WordList"

我将详细解释这些步骤。 例如，一个简单的文本文件可能包含以下单词列表：

cook
 cooked
 cookie
 cookies
 cooking
 cooks
 Copenhagen
 København
 München
 Munich
 ångström

这只有 11 个单词，但如果您的电脑有足够的内存，即使您有一百万个单词，该程序也能正常运行。

您可以使用对象窗口中 Open 菜单下的 从原始文本文件读取 Strings... 命令将该文件读取到一个 Strings（字符串列表） 对象中。生成的 Strings 对象按上述顺序包含 11 个字符串， 您可以通过使用 Inspect（检查） 查看它们来验证这一点。

如果选中该 Strings 对象，您可以点击 To WordList 按钮。 但是，您会收到以下抱怨提示：

String "Copenhagen" not sorted. Please sort first.

该提示意味着这些字符串尚未按照 Unicode 排序顺序进行排序。 所以您点击 Sort，Strings 对象变为：

Copenhagen
 København
 Munich
 München
 cook
 cooked
 cookie
 cookies
 cooking
 cooks
 ångström

这些字符串现在按照 Unicode 排序顺序排列，其中大写字母排在小写字母之前， 复合字符跟在后者后面。

现在点击 To WordList 成功，并且列表中出现一个 WordList 对象。 如果将其保存为文本文件（使用 Save 菜单），您将得到以下文件：

File type = "ooTextFile"
 Object class = "WordList"
string = "Copenhagen
 København
 Munich
 München
 cook
 cooked
 cookie
 cookies
 cooking
 cooks
 ångström\"

请注意，字符串内部出现的任何双引号 (") 都将双写， 就像 Praat 文本文件中所有字符串内部所做的一样。

创建 WordList 文本文件后，您只需通过 Open 菜单中的 从文件读取... 命令 读取此文件即可创建 WordList 对象。

与 Strings 对象相比，WordList 对象的优势在于它不会占用比原始单词列表更多的 内存。这是因为 WordList 被存储为单个字符串： 以换行符分隔的连续字符串列表。
